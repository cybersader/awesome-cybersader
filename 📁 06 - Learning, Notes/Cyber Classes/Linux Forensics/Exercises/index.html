Linux Forensics
Hands-On Exercises

1. Running UAC

Purpose Learn how to customize and run UAC for live incident response.

2. UAC Part 1 - File System Clues

Purpose Detect suspicious processes executing from non-standard locations. Gather more information using collected UAC data.

3. UAC Part 2 - Process Information

Purpose Think you found everything there was to find? Think again!

4. UAC Part 3 - User Accounts

Purpose User account relationships can help us find additional suspicious files and processes.
Audit the user account configuration to look for potential back doors.

5. Bulk Extractor

Purpose It doesn't always have to be about Volatility!
Using bulk_extractor to gather useful leads from memory images.

6. Memory Acquisition

Purpose Acquire memory using different tools and methods. Build Volatility profiles manually.

7. Volatility

Purpose Inspect your memory image using Volatility. Experiment with different plugins.

8. Rootkit!

Purpose If you had a memory image from a system infected with rootkit, how would you investigate? What can you find?

9. Disk Mounting

Purpose Can you master mounting a Linux disk image with RAID, LVM, XFS, and EXT4? Game on!

10. Disk Triage

Purpose We have an image of a compromised web server. Figure out the basic system information for the machine. Quickly start your investigation and find some evil!

11. Timeline Analysis

Purpose Use a file system timeline to gather additional details about our compromised web server. How will this direct your future investigations of this system?

12. Log Analysis

Purpose Logs add more supporting detail about what happened during our compromise. Learn tools to enhance your timeline with critical log information.

13. More Log Analysis

Purpose Web logs answer some burning questions about our compromise. Look at the audit logs from a different system to get a feel for how these can help during an incident.

14. User (Mis)Behavior

Purpose Continuing our web server compromise investigation by looking at some of the user artifacts our attackers left behind.

15. Recover Deleted File

Purpose Can we recover a deleted source code file from EXT file system /tmp using only Open Source tools? Of course we can!

16. XFS Archeology

Purpose Using an inode number from a deleted directory entry to recover a file.
Yes, there will be hex dumps.

17. Honeypot Capstone

Purpose A self-guided adventure as you discover more details from our compromised honeypot image.
